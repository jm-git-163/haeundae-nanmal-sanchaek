# 자동 수집 설계 — 해운대구청·주민센터·정보지가 올라오면 "바로" 앱에

> 목표: 해운대구청과 **18개 행정동 주민센터**, 그리고 **정보지(PDF·이미지)** 에 새 글이 올라오면
> 사람이 손대지 않아도 **자동으로 발견 → 쉬운 말로 변환 → 검수 대기 → 앱 게시**.

**구청 (2)**
- 고시공고 `https://www.haeundae.go.kr/index.do?menuCd=DOM_000000104001002000`
- 새소식·공지사항 (게시판 `BBS_0000038`)

**행정동 주민센터 (18)**
우1·우2·우3 / 중1·중2 / 좌1·좌2·좌3·좌4 / 송정 / 반여1·2·3·4 / 반송1·2 / 재송1·2

> 해운대구청 사이트는 **하나의 게시판 CMS**(`boardId` + `menuCd` 조합)를 쓴다.
> → **어댑터 1개로 20개 게시판을 모두 커버**한다. 동이 늘어도 레지스트리에 한 줄만 추가.

**정보지 (첨부물)**
게시글 첨부의 PDF·이미지(동 소식지, 복지 안내문) → 텍스트 추출 후 같은 흐름을 탄다.

## 2. "바로" 올라오게 하는 법 — 변경 감지

폴링(주기 확인)하되, **새 것만** 집는다.

```
목록 페이지 요청 (If-None-Match / If-Modified-Since 헤더)
   ├ 304 Not Modified  → 아무것도 안 함 (서버 부담 0)
   └ 200 → 목록에서 글 uid 집합 추출
            uid = sha1(상세URL)
            새 uid = 이번 집합 − state.json 에 저장된 지난 집합
            새 uid 만 상세 페이지 수집
```

`data/state.json` (수집기 기억장치)
```json
{ "haeundae-notice": { "etag": "W/\"abc\"", "lastCheck": "2026-07-22T21:00Z",
                        "seen": ["a1b2...", "c3d4..."] },
  "jwa1-dong":       { "etag": null, "seen": [...] } }
```
- `seen` 은 최근 500개만 유지(무한 증가 방지)
- 실패 시 **지수 백오프**(2분→4분→8분), 3회 연속 실패면 운영자 알림

**주기**
| 소스 | 주기 | 이유 |
|---|---|---|
| 구청 고시공고·새소식 | **1시간** | 안전·복지 공지가 빨리 뜬다 |
| 동 주민센터 18곳 | **6시간** | 게시 빈도가 낮음 |
| 정보지(첨부) | 새 글 발견 시 | 이벤트 기반 |
| 축제 OpenAPI | 1일 | 변동 적음 |

## 3. 새 글 하나가 앱에 뜨기까지

```
새 글 발견
 → 상세 본문 추출(제목·본문·게시일·첨부)
 → [선별] 어르신 관련성 0.6 미만이면 폐기        ← 대부분 여기서 버려짐
 → [태깅] audience(연령·동·주제·상황·마감일)
 → [변환] AI: 원문 → 쉬운 3줄 + 이렇게하세요 + 퀴즈
 → pending 큐 (data/pending.json)
 → [검수] 사람이 정확성 확인 → 승인
 → data/feed.json 반영 → 앱이 받아 감(네트워크 우선)
```

**긴급 fast-track** — 폭염·한파·감염병·사기경보·단수/단전 등 **안전 키워드**는
`priority: urgent` 로 표시해 검수 대기열 맨 앞에 놓고, 앱에서는 상단 배너로 노출.
(안전 정보는 늦으면 의미가 없다. 단, **자동 게시는 하지 않는다** — 오정보 위험.)

---

## 4. 정보지(PDF·이미지) 처리

1. 첨부 확장자 판별 → PDF는 텍스트 레이어 추출, 스캔본·이미지는 **OCR**
2. 표·안내문은 문단 단위로 쪼갬
3. 이후는 동일: 선별 → 태깅 → 변환 → 검수

> 정보지는 "버려지는 종이"다. 이걸 **검색되고 읽어주는 디지털 정보**로 바꾸는 것이 이 앱의 강점.

## 5. 구현 산출물 (파일 구조)

```
tools/
  sources.json     # 소스 레지스트리 (구청 2 + 동 18 + API)
  collect.mjs      # 변경 감지 + 새 글 수집 → pending.json
  build_feed.mjs   # 선별·태깅·AI변환 → feed.json (기존)
data/
  state.json       # 마지막으로 본 글 기억
  pending.json     # 검수 대기 큐
  feed.json        # 앱이 받아 가는 최종본
```

실행
```bash
node tools/collect.mjs          # 새 글만 수집
node tools/build_feed.mjs       # 변환·게시
```
GitHub Actions: 구청 소스는 `cron: 0 * * * *`(매시), 동 소스는 6시간 간격.

## 6. 안전하게 — 반드시 지킬 것

- `robots.txt`·이용약관 준수, **요청 간격 ≥ 2초**, 동시요청 1
- `User-Agent` 에 목적·연락처 명시 (기관이 확인 가능하게)
- **원문 URL·게시일·출처 기관 항상 보관·표기**
- 저작물 전문 복제 X → **요지 + 원문 링크**
- 서버 부담 최소화: 304 활용, 실패 시 백오프
- **공식 제휴가 최선**: 해운대구청 홍보협력과 **051-749-4077** 에 데이터 제공(RSS/API) 협의
  → 제휴되면 스크래핑 없이 안정적으로 받는다.

## 7. 새 소식이 "재미있게" 도착하는 방식

수집이 끝이 아니라, 어떻게 **말을 거는지**가 중요하다.

- 🐦 **갈매기가 알려 준다** — "좌동에 새 소식이 왔어요!" (마스코트가 전령)
- 📌 **오늘의 한 가지** — 새 글이 있으면 그날의 한 건으로 승격
- ⏰ **마감 임박** — 신청 마감 D-7이면 상단 배너
- ❓ **바로 퀴즈** — 새 정보에서 문제 하나가 자동 생성 → 맞히면 기억에 남음
- 🔊 **읽어주기** — 새 소식도 눌러서 들을 수 있게
- ✅ **왜 나에게** — "만 65세 이상 · 좌동" 배지로 이유를 밝힘
