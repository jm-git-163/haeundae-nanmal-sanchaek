# 정보시스템 설계 — "어려운 공공정보를 쉽고 재밌게"

> 문제: 부산시·구청·행정복지센터는 좋은 정보를 많이 주지만
> **접근성이 낮고 어려워서 어르신에게 닿지 못한다.**
> 해결: **실시간 확보 → 쉬운 풀이 → 정보지 기반 퀴즈**를 하나의 흐름으로.

이 앱의 진짜 경쟁력(합격 킥)은 콘텐츠 파이프라인입니다. 아래 3단계로 굴립니다.

---

## 1단계 · 실시간 확보 (Ingest)

믿을 수 있는 공식 소스에서 정기적으로 원문을 모읍니다.

| 소스 | 방식 | 주기 |
|---|---|---|
| 부산축제정보 OpenAPI | `busan_api.js` 자동 수집(구현됨) | 매일 |
| 해운대구청 고시·공고 | 공지 목록 확인 → 수기/반자동 등록 | 주 1회 |
| 행정복지센터(주민센터) 알림 | 동별 알림지 확인 | 주 1회 |
| 보건소·질병청 | 폭염·감염병·접종 안내 | 이벤트 발생 시 |
| 복지로/정부24 | 연금·복지 자격·신청 | 월 1회 |

원칙: **공식 출처만**, 확인 안 된 수치·날짜는 올리지 않음.

## 2단계 · 쉬운 풀이 (Transform) — 핵심

어려운 공문 한 건을 **어르신용 카드 한 장**으로 바꿉니다. 한 건(brief)의 표준 형식:

```
{
  cat, title, source, date, until,
  official : "공문 원문 요지 (있는 그대로 · 어려움)",
  lead     : "쉽게 풀면 — 한 줄 (예순다섯 넘으면 지하철 공짜예요)",
  good     : "왜 나에게 좋은지",
  how      : ["이렇게 하세요 ① ② ③ (구체 장소·전화·준비물)"],
  ill      : "삽화 키",
  quiz     : { q, a, tip }   // 이 정보지에서 실제로 알아야 할 것
}
```

앱 구현: `haeundae_media.js`의 `GUIDE`가 이 형식입니다.
- 화면엔 **「쉽게 풀면」을 크게**, **「공식 안내 원문 보기」는 접어서** 보여 줌
  → 어려운 원문과 쉬운 풀이를 **나란히** 두어 변환을 눈으로 확인.

**변환 규칙(쉽게 쓰기 체크리스트)**
1. 한 문장 15자 내외, 존댓말, 숫자는 되도록 한글로("예순다섯").
2. 행정용어 → 생활어("소득인정액"→빼고 "잘 모르면 창구에 물어보세요").
3. 반드시 **행동 1개**로 끝맺기("~하세요").
4. 큰 글씨·삽화 1장(그림 우월효과).

## 3단계 · 정보지 기반 퀴즈 (Recall) — 형식적 금지

퀴즈는 **그 정보지에서 실제로 알아야 할 것**을 묻습니다(이름 맞히기 X).

- 나쁜 예: "동백섬은 무엇일까요?" → 섬 (형식적)
- 좋은 예: "돈을 보내라는 낯선 전화가 오면 먼저 무엇을 해야 하나요?" → **전화를 끊는다**
- 좋은 예: "기초연금은 만 예순다섯이 되기 얼마 전부터 신청하나요?" → **한 달 전**

제약: 정답은 순수 한글(엔진 규칙) → 수치는 "한 달 전 / 예순다섯 살"처럼 한글로.
맞힌 직후 **삽화 + 이렇게 하세요** 즉시 피드백(구현됨) → 기억에 남김.

---

## 데이터 흐름 요약

```
공식 소스 ──(1 수집)──▶ brief 초안 ──(2 쉬운 풀이)──▶ GUIDE 카드(원문+쉬운+실행)
                                          └─(3 퀴즈 생성)──▶ 해운대 퀴즈(정보지 기반)
                                                              └─ 즉시 피드백(삽화+실행팁)
```

한 건을 등록하면 **알아두기 카드 + 퀴즈 문항**이 함께 나오는 것이 목표(1소스→2산출).

## 자동화 로드맵 (수상 후)

- 부산 OpenAPI 실키 연결 → 행사 brief 자동 생성(현재 스냅샷으로 구현).
- 구청/복지로 공지를 LLM으로 "쉬운 풀이 + 퀴즈"까지 초안 생성 → 사람이 검수 후 게시.
- 게시 로그(원문 출처·수집시각·검수자) 보관으로 신뢰성 증빙.

※ 대원칙: **정확하지 않으면 올리지 않는다.** 하나의 오정보가 앱 신뢰를 무너뜨린다.

# 부록 A · AI 적용 방법 (글로벌 오픈소스·논문 조사 결과)

목표: **최신 공공정보를 AI로 쉽고 재미있게** + **검색** + **필요한 사람에게만**.

## A-1. 안전한 소스 수집 (해운대구청 등)
- 대상: 해운대구청 고시·공고, 행복센터 알림, 부산 OpenAPI, 복지로/정부24.
- 안전 규칙: robots.txt·이용약관 준수, 과도한 요청 금지(rate limit), **출처·수집시각 기록**, 개인정보 미수집.
- 문서 파싱: **RAGFlow**(표·레이아웃·인용 강점) 또는 **LlamaIndex**로 공고 PDF/HTML → 구조화. citation-backed(출처 보존).
  - RAGFlow https://github.com/infiniflow/ragflow · LlamaIndex https://www.llamaindex.ai

## A-2. AI로 쉽게 풀기 (플레인 랭귀지)
- 방법: **SimplifyMyText**(arXiv 2504.14223) 식 청중 맞춤 plain-language 변환 프롬프트.
  → 공문 원문 → "어르신 눈높이 3줄 + 이렇게 하세요 + 퀴즈" 자동 초안.
  - https://arxiv.org/html/2504.14223 · JMIR 2025 가독성 향상 https://www.jmir.org/2025/1/e69955
- ⚠️ **핵심 리스크(논문 공통 경고)**: LLM 단순화는 **가독성↑ 대신 사실 왜곡·누락** 위험(readability–accuracy trade-off).
  → 가드레일: ①**공문 원문 보존·병기**(앱에 「공식 안내 원문 보기」로 이미 구현) ②**사람 검수 게이트** ③수치·자격요건은 원문 그대로.

## A-3. 검색 (오프라인·프라이버시) — 구현됨
- 라이브러리 후보: **MiniSearch**(7kB·무의존·퍼지/프리픽스·오프라인)·Fuse.js.
  - MiniSearch https://lucaongaro.eu/blog/2019/01/30/minisearch-client-side-fulltext-search-engine.html
- 현재 앱: **무의존 자체 검색**(소식+알아두기+퀴즈 통합, 기기 안에서만)으로 구현. 데이터가 커지면 MiniSearch로 교체.

## A-4. 프라이버시-우선 로컬 처리
- **LocalRAG**(오프라인·양자화 모델·출처 메타데이터 보호): 어르신 기기에서 문서가 밖으로 안 나감.
  - https://ijcat.com/archieve/volume15/issue1/ijcatr15011003.pdf
- 본 앱 전략: **AI는 서버(빌드타임)에서만** 처리하고, 어르신 기기엔 **정적 결과만** 내려 오프라인·저사양·무추적 유지.

# 부록 B · 차별화 — busan is good / 구청 홈페이지 대비

문제(논문·10x GSA 진단): 공공정보는 **기관 중심(agency-centered)** — 정보는 많지만 흩어져 있고 어렵고 "나에게 해당되는지"를 모른다.

| 축 | busan is good / 구청 홈페이지 | **부산 낱말 산책(차별화)** |
|---|---|---|
| 문장 | 공문·행정용어 | **AI로 3줄 쉬운말 + 이렇게 하세요** |
| 찾기 | 메뉴 깊숙이 | **자연어 검색**(지하철·연금·사기) |
| 대상 | 전체에게 동일 | **나에게 필요한 것만**(개인화) |
| 정확성 | 원문만 | 쉬운말 **+ 원문 병기 + 검수** |
| 지속 | 한 번 보고 이탈 | **퀴즈·갈매기로 재미**(재방문) |
| 도달 | 스스로 찾아야 | **매일 소식이 먼저 다가옴** |

## B-1. "필요한 사람에게 필요한 정보" (개인화, 무PII) — 다음 구현
근거: USA.gov **Benefits Finder**(10x/GSA)·**DiscoverSeniorBenefits** — 몇 개 질문으로 해당 없는 제도를 걸러 **맞춤 목록**을 만든다. SSN 등 **개인정보 미수집·익명**.
- https://10x.gsa.gov/investments/01-investments-in-benefits-eligibility/ · https://discoverseniorbenefits.com/
- 앱 적용안: 설정에서 **관심 주제(연금·건강·교통·안전·나들이)·연령대**만 고르면(기기 저장, 전송 없음),
  알아두기·소식·퀴즈를 **내게 맞는 순서로** 정렬/필터. "나에게 해당되는 지원" 배지.
  → 자격요건은 규칙(rule)로 인코딩(예: 만 65세 이상 → 지하철 무료·기초연금 노출).

## B-2. 데이터 흐름(확장판)
```
공식 소스 ─수집(RAGFlow/LlamaIndex, 출처보존)→ 원문
   → AI 쉬운말(SimplifyMyText식) + 퀴즈 초안 → 사람 검수(정확성 게이트)
   → briefs 게시 → [앱] 오프라인 검색 · 개인화 필터 · 쉬운 카드 · 퀴즈
```

## B-3. 구현 현황
- ✅ 쉬운말+원문 병기, 정보지 기반 퀴즈, 즉시 피드백(삽화+실행법)
- ✅ **오프라인 통합 검색**(소식+알아두기+퀴즈, 기기 안)
- ⬜ 개인화 필터(관심주제·연령, 무PII) — 다음
- ⬜ AI 수집→쉬운말→퀴즈 자동 초안 파이프라인(RAGFlow+LLM) + 사람 검수
