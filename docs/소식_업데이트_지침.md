# 해운대 소식 — 업데이트 운영 지침

> 이 앱의 신뢰도는 **"믿을 수 있는 정보를, 제때, 꾸준히"** 올리는 데서 나옵니다.
> 소식 하나하나가 어르신의 실제 결정(신청·예방·참여)에 영향을 주므로, 아래 규칙을 반드시 지킵니다.

## 1. 어디서 가져오나 (신뢰 출처만)

| 유형 | 공식 출처 | 확인처 |
|---|---|---|
| 행사·축제 | 부산축제정보 OpenAPI | https://www.data.go.kr/data/15063500/openapi.do |
| 구정 공지 | 해운대구청 고시·공고 | https://www.haeundae.go.kr |
| 동 단위 알림 | 각 주민센터 게시판·알림지 | 해운대구 행정복지센터 |
| 복지·연금 | 보건복지부·국민연금공단·정부24 | https://www.gov.kr |
| 건강·감염병 | 질병관리청·해운대구보건소 | https://www.kdca.go.kr |
| 안전·범죄예방 | 해운대경찰서·부산경찰청 | 관할 공지 |

❌ 개인 블로그·카페·확인 안 된 SNS는 출처로 쓰지 않는다.

## 2. 올리는 규칙 (5원칙)

1. **출처 명기**: 모든 소식에 `source`(발표 기관)를 적는다. 화면에 "출처 · 기관명"으로 노출된다.
2. **날짜 명기**: `date`(올린/발표한 날)를 적는다. 최신순으로 자동 정렬된다.
3. **자동 만료**: `until`(마감일)을 적으면 그 날이 지나는 즉시 화면에서 사라진다 → 철 지난 정보가 남지 않는다.
4. **검증된 것만**: 확인되지 않은 수치·날짜·금액은 올리지 않는다. 애매하면 "가까운 주민센터에 문의" 식으로 안전하게 쓴다.
5. **쉬운 말**: 어르신이 한 번에 이해하도록 짧고 큰 글씨의 존댓말로. 한 소식 3~4문장 이내.

## 3. 어떻게 올리나 (파일 편집)

`js/notices.js`의 `R.NOTICES` 배열에 항목을 추가한다. 형식:

```js
{
  cat: '복지',                    // 복지 | 행사 | 안전 | 건강 | 생활
  title: '노인복지관 여름 무료강좌',
  body: '해운대 노인종합복지관에서 스마트폰·건강체조 강좌를 무료로 엽니다. 신청은 복지관으로.',
  source: '해운대노인종합복지관',   // 출처(공식 기관)
  date: '2026-07-14',            // 올린 날 (필수)
  until: '2026-08-30'            // 마감일 (선택, 지나면 자동 숨김)
}
```

배포 반영: 파일 저장 → `sw.js`의 `CACHE` 값 한 단계 올림 → (공모전용 별도 Vercel 프로젝트에) 배포.

## 4. 갱신 주기

- **매주 1회 이상** 정기 점검(권장: 월요일).
- 만료된 소식 정리 + 새 공지 반영 + 계절 이슈(폭염·한파·명절) 선반영.
- 큰 행사(바다축제·불꽃축제 등)는 2~3주 전 미리 올린다.

## 5. 자동화 로드맵 (수상 후 확장)

- **부산축제정보 OpenAPI 연동**: 행사 소식을 사람이 아닌 API에서 매일 자동 수집 → "AI가 공공데이터로 소식을 스스로 갱신"으로 발표. (실현가능성·AI 존재감 가점)
- **소식 → 퀴즈 자동 생성**: 새 소식이 올라오면 관련 해운대 퀴즈 문항을 규칙/LLM으로 자동 생성.
- 이때 원문 출처와 갱신 시각을 로그로 남겨 신뢰성을 증빙한다.
