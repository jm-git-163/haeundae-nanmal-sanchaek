# 정보 전달 설계 — 웹의 최신 정보를 "그 사람에게" 쉽고 재미있게

> 이 앱의 존재 이유: **부산이즈굿·구청 홈페이지에 흩어진 정보를 모아, 나에게 해당되는 것만, 쉽게, 재미있게.**
> 기존 서비스는 *모두에게 같은 목록*을 준다. 우리는 *한 사람에게 맞는 몇 개*를 준다.

핵심 흐름 (7단계)

```
0.소스  →  1.수집  →  2.정규화·중복제거  →  3.선별(어르신 관련성)
        →  4.대상자 태깅  →  5.쉬운 변환(AI)  →  6.매칭·랭킹  →  7.전달
```

## 1. 수집 — 소스 어댑터 레지스트리

소스마다 어댑터 하나. 새 기관 추가 = 어댑터 하나 추가.

```js
{ id:'haeundae-notice', name:'해운대구청 공지', type:'html'|'rss'|'api',
  url:'...', trust:'official',      // official | press | partner
  cadence:'daily',
  parse(res) -> RawItem[] }
```

## 2. 정규화 · 중복 제거

```js
RawItem { uid, source, sourceUrl, publishedAt, title, body, region, attachments[] }
```
- `uid` = sha1(sourceUrl + title) → 재수집해도 중복 안 쌓임
- 같은 사안이 시·구에 동시 게시되면 **제목 유사도 ≥0.85** 는 하나로 병합(출처는 둘 다 보관)

## 3. 선별 — "어르신에게 쓸모 있는가" (정보 과잉 해결의 핵심)

구청 공지의 대부분은 어르신과 무관하다(입찰·인사·조례). **먼저 버린다.**

- **제외 규칙**: 입찰·낙찰·계약·용역·수의계약·인사발령·채용·의회·조례안·감사·공유재산
- **포함 규칙(가점)**: 어르신·경로·노인·연금·복지·건강·접종·검진·무료·신청·지원금·급식·경로당·복지관·폭염·한파·감염병·보이스피싱·교통·버스·지하철·문화강좌·일자리·쉼터
- **AI 판정**: "65세 이상 주민이 이 공지를 읽고 **실제로 할 수 있는 행동**이 있는가? 0~1 점수와 이유" → **0.6 미만 폐기**

> 이 단계가 없으면 앱은 그냥 또 하나의 공지 목록이 된다.

## 4. 대상자 태깅 — 매칭의 뼈대

살아남은 항목에 **누구를 위한 정보인지** 태그를 붙인다(AI 초안 + 사람 검수).

```js
audience: {
  age:      { min:65, max:null },          // 연령 조건
  region:   ['해운대구'] | ['좌동','중동'], // 구 전체 또는 특정 동
  topics:   ['복지','건강'],                // 복지·건강·안전·생활·나들이·행사
  situations:['독거','거동불편','기초연금대상','치매걱정','손주돌봄'],
  deadline: '2026-08-31'                   // 신청 마감(있으면)
}
```

## 5. 쉬운 변환 — AI가 공문을 어르신 말로

한 건 = **원문 + 쉬운 풀이 + 실행법 + 퀴즈**

```js
{ official:'공문 원문 요지(그대로)',
  lead:'쉽게 한 줄(15자 내외, 숫자는 한글)',
  good:'왜 나에게 좋은지',
  how:['① 챙길 것','② 갈 곳','③ 문의 전화'],
  quiz:{ q, a, tip } }
```
**정확성 가드레일(필수)**: ①원문 병기 ②수치·자격·날짜는 원문 그대로 ③사람 검수 전엔 `pending` ④읽기 수준 초등 5~6학년 목표.

## 6. 매칭·랭킹 — "나에게 맞는 몇 개"

기기 안에만 저장되는 프로필(개인정보 없음):

```js
me = { ageBand:'65+', dong:'좌동', topics:['복지','안전'], situations:['독거'] }
```

**점수 공식**

```
score = 2.0·연령적합 + 1.5·지역적합 + 1.2·관심적합 + 1.0·상황적합
      + 0.8·신선도(최근 7일 내 가중) + 1.5·마감임박(D-7 이내) + 0.5·실행가능(how 있음)
```
- 임계 미만은 **아예 안 보여 준다**(과잉 차단)
- 상위 N개만 노출, 나머지는 검색으로만

**"왜 나에게 왔는지" 설명 배지** — 신뢰와 이해를 동시에 올린다.
> `✅ 만 65세 이상 · 좌동 · 복지` / `⏰ 신청 마감 7일 전`

기존 서비스와의 결정적 차이: 부산이즈굿은 *모두에게 같은 목록*, 우리는 *나에게 해당되는 것 + 그 이유*.

## 7. 전달 — 쉽고 재미있게

| 장치 | 목적 | 상태 |

| 매칭 목록 + 왜 나에게 배지 | 관련성·신뢰 | 배지 일부 구현 |
| ⏰ **마감 임박 D-day** | 행동 유도 | 설계 |
| 🔊 **읽어주기** | 글 못 읽는 분(60세+ 71%) | ✅ 구현 |
| ❓ **정보지 기반 퀴즈 + 즉시 피드백** | 기억·재미 | ✅ 구현 |
| 🐦 갈매기가 전해주는 연출 + 연속일 | 친근·지속 | 일부 구현 |
| 🔎 검색 | 필요할 때 찾기 | ✅ 구현 |
| 출처·발표일·확인일·원문보기 | 신뢰 | ✅ 구현 |

## 8. 운영 루프 · 완성도 측정

```
매일 06시 자동수집 → 선별 → 태깅 → AI 변환 → PR → 사람 검수 → feed.json 배포
```

성과 지표(이게 "완성도"의 증거):
- **도달**: 매칭된 항목을 실제로 연 비율
- **이해**: 그 정보로 낸 퀴즈 정답률
- **행동**: 「이렇게 하세요」 펼침률·전화 걸기
- **지속**: 연속 접속일

※ 대원칙: **정확하지 않으면 올리지 않는다. 관련 없으면 보여 주지 않는다.**
